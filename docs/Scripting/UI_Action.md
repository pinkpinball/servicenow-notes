# UI Action

## Purpose:

- Add buttons / actions to UI

## Runs:

- Client, Server, or Both
- On click

## Use When:

- User-triggered actions
- NOT for background enforcement

## Touches:

- Tables: current table
- APIs: g_form, current

## Common Mistakes:

- Mixing client/server logic incorrectly
- No condition guarding visibility
